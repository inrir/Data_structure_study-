BIG - O
프로그램이 동작하는 것이 중요하다면 스킵하되.
성능적으로 우월을 판단하는 경우, BIG-O를 통해 성능차이를 분석해야한다.

대략적인 숫자를 세는 것(전반적인 추세를 계산.).    
O(f(n)), 입력과 실행의 관계.

O(2n + 50) => O(n)
O(n^2 + 5n + 51324) => O(n^2)

시간복잡도.
: 입력이 커질수록 알고리즘의 실행 속도가 어떻게 바뀌는지 분석.

공간복잡도. 
: 사용되는 메모리. 저장될 공간의 복잡도를 탐색하는 것이다.
  정의된 변수에 입력이 어떤식으로 들어옴에 따라서 정하는 것이다...

Logarithms
O(logN)
로그함수는 지수함수와 짝이다.
*탐색 알고리즘, 효율적인 정렬 알고리즘, 재귀에서 사용됨.

  1 < log N < N < N log N < N^2
